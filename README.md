# NewAPI Monitor Plugin

面向 TRSS-Yunzai 的多站 New API 余额与分组倍率监控插件。支持主人专用 Web 配置、实时查询、低余额状态提醒，以及所选多个分组的倍率变动提醒。

## 安装

将整个目录放到：

```text
TRSS-Yunzai/plugins/newapi-monitor-plugin
```

重启 TRSS-Yunzai。插件使用 TRSS-Yunzai 已包含的 `express` 和 `yaml` 依赖，无需在插件目录单独安装依赖。

## 命令

以下命令仅机器人主人可用：

- `#NewAPI监控登录`：启动 Web 配置服务并生成一个 5 分钟内有效、只能使用一次的登录链接。
- `#NewAPI余额`：立即查询所有已启用站点的余额和全部所选分组倍率，不改变监控告警状态。

Web 服务默认监听 `127.0.0.1:25095`。

## 凭据

插件不使用 New API 账号密码登录。每个站点需要填写：

1. New API 个人设置中生成的“系统访问令牌”。
2. 与该令牌所属账号匹配的 New API 用户 ID。

插件以以下请求头调用用户接口：

```http
Authorization: Bearer <系统访问令牌>
New-Api-User: <用户 ID>
```

连通性测试会读取：

```text
GET /api/user/self
GET /api/user/self/groups
```

系统访问令牌保存在 `config/config.yaml`，Web 接口只返回“已配置”标记，不会把原令牌回传给浏览器。测试新令牌时，令牌会在同源 HTTPS/HTTP 请求中发送；跨域重定向会被拒绝。

## 站点配置

在 Web 页面中添加站点，并依次填写：

- 站点名称
- 站点 URL
- New API 用户 ID
- 个人系统访问令牌
- 额度换算值
- 余额预警阈值
- 需要监控倍率的一个或多个分组

点击“测试连通性并获取分组”后，插件会同步获取账户信息和分组倍率。测试成功后可用复选框选择一个或多个倍率监控分组。

站点 URL 会去除查询参数、锚点、末尾斜杠，以及误粘贴的 `/log`、`/logs`、`/api`、`/dashboard`、`/console` 等页面路径。对于部署在子路径下的 New API，插件会先尝试保留部署路径，再回退到站点根地址。

余额显示值的计算方式为：

```text
余额 = New API quota / 额度换算值
```

不同站点的额度换算规则可能不同，请按站点实际规则设置。

## 仪表盘

主人登录 Web 页面后默认进入总览。总览会显示全部已配置站点数量、成功获取余额的站点范围、合计余额，以及每个站点的当前余额、近 1 小时和近 24 小时请求量与余额消耗。

统计只使用个人系统访问令牌可访问的用户接口：

```text
GET /api/user/self
GET /api/log/self
GET /api/log/self/stat
```

数据口径：

- 当前余额与总余额：账户 `quota / 额度换算值`。
- 请求量：指定时间窗内 `type=2` 消费日志列表返回的 `data.total`。
- 消耗余额：指定时间窗内日志统计返回的 `data.quota / 额度换算值`。
- `/api/log/self/stat` 返回的 `rpm` 在 New API 后端固定为最近 60 秒请求数，因此插件不会用它推算 1 小时或 24 小时请求量。

`/api/log/search` 需要管理员权限，本插件不会使用该接口。某个站点不支持个人日志接口或禁止当前用户读取日志时，该站余额仍会显示，1 小时和 24 小时栏会明确显示接口错误，不会生成估算值。

## 提醒规则

插件默认每分钟检查一次所有已启用站点，并限制并发请求数量。

余额提醒：

- 首次发现余额低于阈值时提醒一次。
- 余额持续低于阈值时不重复提醒。
- 余额恢复到阈值或以上时静默复位。
- 之后再次跌破阈值时重新提醒。

分组倍率提醒：

- 每个所选分组首次成功获取的倍率只用于建立各自基线，不提醒。
- 任一所选分组的倍率发生变化时，都会单独提醒并更新该分组基线。
- 新增选择只为新分组建立基线，不会重置仍在监控的其他分组。
- 某个所选分组暂时不存在时记录错误，但不会覆盖该分组上一次倍率基线，也不影响其他分组继续监控。

旧版单分组配置会在读取时自动迁移为多分组数组，无需手动修改配置文件。

告警通过 `Bot.sendMasterMsg` 发送给机器人主人。站点连接失败会记录在 Web 运行状态和 Yunzai 日志中，不会每分钟发送失败消息刷屏。

## 远程访问

默认配置只允许本机访问：

```yaml
web:
  host: "127.0.0.1"
  port: 25095
  publicUrl: "http://127.0.0.1:25095"
```

需要远程配置时，推荐保持 `host: "127.0.0.1"`，使用 Nginx、Caddy 等反向代理到 `127.0.0.1:25095`，并将 `publicUrl` 改为外部 HTTPS 地址。不要直接把管理端口暴露到公网。

修改运行时配置文件：

```text
plugins/newapi-monitor-plugin/config/config.yaml
```

修改监听地址或端口后需要重启 TRSS-Yunzai。

## 数据与备份

运行时文件不会提交到 Git：

```text
config/config.yaml
data/state.json
```

`config/config.yaml` 包含系统访问令牌。备份、迁移和日志采集时应按敏感文件处理，不要发到群聊、Issue 或公开仓库。`data/state.json` 保存余额告警复位状态和倍率基线，重启后会继续沿用，避免重复提醒。

在 TRSS-Yunzai 根目录运行：

```powershell
node --test plugins/newapi-monitor-plugin/test/*.test.js
```
